# SQL Analyst Tool

The SQL analyst tool turns a natural-language analytics question plus an analytical context into executable federated SQL and a normalized result payload.

## Responsibilities

1. Ground the question using the selected analytical context.
2. Generate a canonical PostgreSQL query over the dataset aliases exposed by that context.
3. Validate and preview-limit the query before execution.
4. Execute the query through the federated dataset path and return normalized rows.

## Analytical Contexts

The tool accepts one analytical context per run:

- Dataset assets, which expose one or more datasets directly.
- Semantic model assets, which govern datasets but still execute through federation.

Each context includes:

- Asset metadata such as type, name, description, and tags.
- Dataset bindings with dataset name, SQL alias, source/storage kind, and columns.
- Optional semantic fields such as dimensions, measures, metrics, and relationships.

## Request Contract

| Field | Description |
| --- | --- |
| `question` | Natural-language analytics request. |
| `conversation_context` | Optional follow-up context from earlier turns. |
| `filters` | Optional structured filters to apply in the generated SQL. |
| `limit` | Preview row limit enforced before federated execution. |
| `semantic_search_result_prompts` | Optional grounding hints when semantic or vector search is available. |

## Response Contract

| Field | Description |
| --- | --- |
| `analysis_path` | Whether the selected asset was a `dataset` or `semantic_model`. |
| `execution_mode` | Always `federated`. |
| `asset_type` | Asset type selected by the analyst agent. |
| `asset_name` | Human-readable name of the selected asset. |
| `selected_datasets` | Dataset bindings used to ground and execute the query. |
| `sql_canonical` | SQL generated by the language model in PostgreSQL dialect. |
| `sql_executable` | SQL after limit enforcement. |
| `result` | Normalized tabular result payload. |
| `error` | Error message when SQL generation or execution fails. |

## Execution Flow

1. Render the selected analytical context into the SQL generation prompt.
2. Call the language model to produce a single PostgreSQL `SELECT`.
3. Validate the generated SQL with `sqlglot`.
4. Apply preview limits with `enforce_preview_limit`.
5. Dispatch the executable SQL through the federated query executor.
